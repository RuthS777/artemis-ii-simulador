# Políticas de Git y Ramificación

Para cumplir con las normas del profesor Bladimir Báez, acordamos:
1. Nadie programa directo en la rama `main`.
2. Cada tarea se hace en una rama separada llamada `feature/nombre-tarea`.
3. Cuando se termine una tarea, se avisa al equipo y un compañero debe revisar y aprobar el cambio en menos de 24 horas (Pull Request).
